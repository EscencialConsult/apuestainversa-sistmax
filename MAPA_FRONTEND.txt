■ PUNTO DE ENTRADA
================================================================

■ SERVICIOS / API  →  src/services/api/
================================================================

Todos los servicios son MOCK actualmente. Cada uno simula las
respuestas que el backend real deberá implementar.
El tenantId es SIEMPRE el primer filtro en cada función.

ARCHIVO                          TIPO        QUÉ HACE / ENDPOINTS QUE SIMULAN
─────────────────────────────    ────────    ──────────────────────────────────────────
services/api/
  apiClient.js                   JS          Instancia de Axios configurada.
                                             Interceptor para inyectar Bearer token.
                                             Base URL desde VITE_API_URL.

  arcaService.js                 JS          GET /api/arca/verificar/:cuit
                                             Verificación ARCA/AFIP de proveedores.
                                             Simula timeout y modo contingencia.

  catalogosService.js            JS          GET /api/:tenantId/config/modalidades
                                             GET /api/:tenantId/config/rubros
                                             Datos de configuración del municipio.

  adjudicacionService.js         JS          GET plantillas de contrato y orden.
                                             POST firmar etapa del circuito.
                                             Genera texto de los documentos legales
                                             con el nombre INVERSA.Bid.

  aprobacionService.js           JS          GET jerarquías de aprobación.
                                             GET/POST expedientes pendientes de firma.

  auditoriaService.js            JS          GET /api/:tenantId/auditoria (con filtros)
                                             POST evento externo (logEventoExterno).
                                             Genera 50+ eventos mock con fechas reales.

  expedienteService.js           JS          GET /api/:tenantId/compras/:id
                                             Detalle completo del expediente (lances,
                                             proveedores, aprobación, recepción).

  portalService.js               JS          GET /portal/:tenantId/procesos (públicos)
                                             GET /portal/:tenantId/documentos
                                             Solo expone PUBLICADO y ADJUDICADO.

  adjudicacionService.js         JS          (ver arriba)
